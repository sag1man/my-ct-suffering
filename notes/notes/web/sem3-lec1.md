# Введение в HTTP
HTTP - Клиент-серверный протокол прикладного уровня
До http3 использовался TCP, начиная с http3 - UDP через QUIC 

## URI и URL
##### Определения
- Universal Resource Identifier (URI) - последовательность символов, идентифицирующая абстрактный или физический ресурс. 
- Universal Resource Locator (URL) - стандарт который используются, чтобы определить местоположение ресурсов, обеспечивая абстрактную идентификацию расположения ресурса

##### Особенности в HTTP
- Символы которые непредставимы в ASCII кодируются процентным кодированием
- Схема и хост - регистро-независимые, а остальные части регистро-зависимые

## HTTP Запросы
![[web-sem3-lec1-img2.png]]

- Запрос начинается с метода, основные это:
	- **GET** (получить данные) 
	- **POST** (отправить данные)
- Дальше идет относительный URL, в примере это корень
- Потом указывается версия протокола
- Дальше идут разные заголовки в произвольном порядке, среди них есть один обязательный:
	- **Host** - указывает hostname ресурса, нужно это для того, если один сервер обслуживает несколько ресурсов
